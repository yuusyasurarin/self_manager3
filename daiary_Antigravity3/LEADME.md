# 日記＆目標管理Webアプリ: 学習テキスト (LESSON.md)

本ドキュメントは、「日記＆目標管理Webアプリ」の構築を通じて、モダンなWeb開発の基礎となる理論を学び、GoogleスプレッドシートとGoogle Apps Script (GAS) を用いた完全無料のバックエンドを構築するための手順書です。

---

## 1. Web技術のコア理論

### 1-1. 非同期通信の仕組み（fetch API, Promise, async/await）

アプリの画面をリロードせずに、裏側でサーバー (GAS) とデータのやり取りを行う技術が**非同期通信**です。

- **シングルスレッドと非同期**
  JavaScriptは「シングルスレッド（1本道）」で動く言語です。そのため、通信などの時間のかかる処置を「同期的（順番待ち）」に処理してしまうと、通信が終わるまで画面操作がフリーズしてしまいます。そこで、通信はバックグラウンドに任せて別の処理を進め、結果が戻ってきたら続きを行う「非同期処理」が必須となります。
- **Promise（プロミス）**
  非同期処理の「将来の結果の約束」を表すオブジェクトです。状態として「保留中 (Pending)」「成功 (Fulfilled)」「失敗 (Rejected)」を持ちます。
- **async / await**
  Promiseをより直感的に、まるで同期的であるかのようにコードを書くための機能です。関数に `async` をつけると必ずPromiseを返すようになり、その中で `await` を使うと、指定した非同期処理の完了（通信の終了など）を「待つ」ことができます。これにより、コールバック地獄を防ぎ、読みやすいコードになります。
- **fetch API**
  ブラウザに標準で組み込まれている、ネットワーク越しにリソースを取得するための最新APIです。`fetch('URL')` を呼ぶとPromiseが返り、結果のResponseオブジェクトからJSONデータをパースするなどの処理を行います。本アプリでもGASとの通信に全面的に利用しています。

### 1-2. DOM操作とイベント駆動モデル

Webページは単なる文書ではなく、プログラムから操作可能な「オブジェクトのツリー構造」です。これを **DOM (Document Object Model)** と呼びます。

- **DOM操作**
  HTMLの各要素（タグ）はJavaScriptからはNodeとして扱われます。`document.getElementById()` や `document.createElement()` などを用いて要素を取得・生成し、テキストやスタイルを動的に書き換えることで、本アプリのような Single Page Application (SPA) が実現されています。画面遷移を起こさず、DOMを直接書き換えることで高速な操作感を生み出します。
- **イベント駆動モデル**
  「ユーザーが〇〇した時」にプログラムが動く仕組みです。例えばボタンがクリックされた、画面がスクロールされたなどの「イベント」に対して、`addEventListener` で「イベントハンドラ（関数）」を事前に登録しておきます。JavaScriptのプログラムは普段は待機しており、「クリック」などのイベントが発火した瞬間だけ反応して対応する処理（DOMの更新や通信など）を行います。

### 1-3. セキュリティとCORS（Cross-Origin Resource Sharing）の概念

インターネット上では悪意のあるサイトからの攻撃を防ぐため、ブラウザはデフォルトで「**同一オリジンポリシー (Same-Origin Policy)**」という制限を設けています。
オリジンとは「スキーム (http/https)」「ホスト (ドメイン)」「ポート」の組み合わせです。たとえば、`https://your-app.com` から `https://script.google.com` (GAS) にAJAX通信を行うことは、異なるオリジンへのアクセスとなるため原則ブロックされます。

- **CORSとは**
  この制限を安全に解除するための仕組みが **CORS** です。サーバー側（今回はGAS側）が、HTTPヘッダーに `Access-Control-Allow-Origin: *` または特定のドメインを含めることで、「このリクエストは許可されている」とブラウザに伝え、通信を成功させます。
- **GASにおけるCORS**
  GASの `doGet` や `doPost` を用いたWebアプリは、`ContentService.createTextOutput()` を用いてレスポンスを返しますが、MIMEタイプを `JSON` 等に正しく指定し特定の書き方をすることで自動的にCORSのリダイレクトと許可処理が行われます。ただGASの仕様上、初期通信は常にリダイレクト（302 HTTP status）を返すため、fetch APIを使う際はリダイレクトに追従する設定（デフォルトで追従します）を利用します。

## 2. バックエンド構築手順（Google スプレッドシート + GAS）

この手順に従い、本アプリ専用のクラウドデータベースとAPIエンドポイントを構築します。

### ステップ 1. Google スプレッドシートの作成
1. ご自身のGoogleアカウントでログインし、Googleドライブ等から新規の**Google スプレッドシート**を作成します。
2. スプレッドシート名は何でも構いません（例：「My Diary & Goals App Data」）。
3. 画面下部のシートタブ部分で、以下の4つのシート名に書き換えて追加・作成してください。
   ※スペルが一文字でも違うとエラーになるため注意してください。
   - `MonthlyGoals`
   - `WeeklyGoals`
   - `DailyRecords`
   - `Tasks`
   ※シートのヘッダー列（1行目）はプログラム側で扱わない設計としていますが、ご自身で「年月」「目標」などのメモを書いておいても問題ありません。

### ステップ 2. Google Apps Scriptへのアクセス
1. 作成したスプレッドシートの上部メニューから **「拡張機能」 > 「Apps Script」** をクリックします。
2. 新しいタブにGASの開発画面が開きます。
3. エディタ内にある初期の `function myFunction() {}` をすべて消去します。
4. 本アプリのワークスペースに生成された `Code.gs` ファイルの内容を全てコピーし、このGASエディタに貼り付けます。
5. 「プロジェクトを保存」（フロッピーディスクのアイコンまたは Ctrl+S / Cmd+S）をクリックします。

### ステップ 3. Webアプリとしてのデプロイ（URLの発行）
1. GAS画面の右上にある **「デプロイ」** ボタンをクリックし、**「新しいデプロイ」** を選択します。
2. 「種類の選択」の左側にある歯車アイコン(⚙) をクリックし、**「ウェブアプリ」** にチェックを入れます。
3. 以下の通りに設定します：
   - **説明**: （空欄でも可。例：「初回デプロイ」）
   - **次のユーザーとして実行**: **「自分（あなたのアカウントメールアドレス）」**
   - **アクセスできるユーザー**: **「全員」** （これが必須です。全員としないとアプリからアクセスできません。※Googleアカウント認証画面が出たりCORSエラーの原因になります。）
4. **「デプロイ」** をクリックします。
5. （初回のみ）「アクセスを承認」というポップアップが出ます。
   - 自分のGoogleアカウントを選択します。
   - 「Google が確認していません」という警告画面が出ますが、左下の **「詳細」 > 「○○（プロジェクト名）に移動（安全ではないページ）」** をクリックします。
   - 許可を求められるので「許可」をクリックします。
6. デプロイが完了すると、「**ウェブアプリの URL**」が発行されます。（`https://script.google.com/macros/s/XXXXXX/exec` という形式）
7. このURLの **「コピー」** をクリックして控えておきます。このURLが本アプリの専用APIエンドポイントになります。

### ステップ 4. フロントエンド（アプリ側）との連携設定
1. VSCodeなどのエディタで、ワークスペース内の `app.js` を開きます。
2. ファイル上部付近にある定数設定で、`USE_MOCK` と `GAS_API_URL` の箇所を見つけます。
3. 以下のように書き換えてください。
   ```javascript
   // テスト用モックを利用するかどうか (本番連携時は false にする)
   const USE_MOCK = false;

   // 先ほどコピーしたGASのウェブアプリURLを貼り付ける
   const GAS_API_URL = "https://script.google.com/macros/s/XXXXXX/exec";
   ```
4. 保存後、ブラウザで `index.html` またはご自身のサーバー（GitHub Pagesなど）にアクセスし、アプリから入力したデータがスプレッドシートに書き込まれるか確認してください。

以上で、完全無料・サーバーレスのバックエンドと連携したオリジナルWebアプリの開通となります。お疲れ様でした！
